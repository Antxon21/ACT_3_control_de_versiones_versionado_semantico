## Pasos de la práctica: 
1. Sube la Práctica3 a github como primera versión en la rama master.
2. Crea una rama nueva para desarrollo, y actualiza la versión.
3. Realiza 2 refactorizaciones.
4. Actualiza la rama master con la nueva versión estable.
5. Realiza 2 refactorizaciones más en la rama de desarrollo desde la
versión anterior.
6. Sube a máster lo anterior y actualiza la versión.

- Realiza el control de versiones con versionado semántico.
- Aplica comentarios en formato JavaDoc.
- Utiliza Markdown para explicar lo que has hecho en la práctica.

## Explicación de cada punto: 
1. En el primer punto, agregamos la práctica 3 a github como primera versión en la rama master.
2. En el segundo punto, creamos una nueva rama llamada Development que es donde vamos a trabjar con el código y en donde iremos realizando los commits.
3. En el tercer punto, realizamos las 2 refactorizaciones y en ellas agregamos los comentarios en JavaDoc con los parámetros correspondidos.

   - Aparte lo subimos a github y lo actualizamos con la versión correspondida
4. En el cuarto punto lo actualizaremos/fusionaremos a la rama master con la versión establa
5. En el quinto punto hacemos la 2 refactorizaciones restantes en la rama de Development.
6. Por último punto (sexto) agregamos todo lo anterior a la rama master y le agregamos la nueva versión
